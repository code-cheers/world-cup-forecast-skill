# Forecast Reports

Daily generated reports are written here by `.github/workflows/daily-world-cup-forecast.yml`.

- `latest.md` contains the newest generated report and embeds `assets/latest-dashboard.svg`.
- `daily/YYYY-MM-DD.md` contains dated snapshots and embeds the matching dated dashboard.
- `assets/latest-dashboard.svg` contains the newest visual dashboard.
- `assets/YYYY-MM-DD-dashboard.svg` contains dated dashboard snapshots.
- `assets/flags/` contains cached SVG flags used inside generated dashboards.
- `analysis/latest.md` contains the newest Codex-written model analysis.
- `analysis/YYYY-MM-DD.md` contains dated Codex-written analysis snapshots.

The workflow runs at 08:00 Asia/Shanghai using GitHub Actions. Reports are generated from public tournament data, the transparent heuristic model, and optional market signals when a market JSON file is provided.

Daily report Markdown is filled from `assets/daily-report-template.md` in the repository root. The intended GitHub reading order is dashboard first, short conclusions second, auditable tables third.

The model analysis is generated by a local Codex App automation, not by OpenAI API calls in GitHub Actions. It requires the local machine and Codex app automation runtime to be available at the scheduled time.
