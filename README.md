# World Cup Forecast Skill

这个 skill 用来获取、汇总和预测 FIFA 男足世界杯数据。它面向中文和英文问题，例如“预测世界杯排名”、“世界杯下一轮预测”、“世界杯冠亚军”、“世界杯球员进球数”、“World Cup forecast”等。

## 作用

- 查询历届男足世界杯冠军、亚军、季军和第四名。
- 汇总当前世界杯赛程、已完赛结果、小组积分榜和当前射手榜。
- 预测下一轮未赛比赛的胜平负概率。
- 通过蒙特卡洛模拟预测最终冠军、亚军、第三名、第四名概率。
- 输出最可能的冠亚季军组合，而不是把单项最高概率硬拼成一个可能重复球队的组合。

这个 skill 的目标是给出透明、可复现的方向性预测，不是博彩建议，也不是保证结果。

## 快速使用

在 skill 目录下运行：

```bash
python scripts/worldcup_forecast.py history --since 1926 --positions 4 --format markdown
python scripts/worldcup_forecast.py current --season 2026 --format markdown --lang zh
python scripts/worldcup_forecast.py predict --season 2026 --target next-round --format markdown --lang zh
python scripts/worldcup_forecast.py predict --season 2026 --target final --runs 2000 --seed 20260614 --format markdown --lang zh
```

英文输出可以加 `--lang en`。需要机器处理时使用 `--format json`。

## 数据来源

默认采用免费公开数据源：

| 数据源 | 用途 |
| --- | --- |
| OpenFootball `worldcup.json` | 当前赛程、比分、小组、场地、已记录进球者 |
| Fjelstul World Cup Database | 历史最终排名、历史强队权重 |
| World Football Elo | 当前国家队公开 Elo 强度，默认启用 |
| FIFA/Coca-Cola Men's World Ranking | 可选官方排名和积分，使用 `--use-fifa-ranking` 启用 |

OpenFootball 数据不是秒级直播源，可能有人工更新延迟。实时阵容、xG、射门、赔率、攻势图和完整球员事件通常需要带 API key 的数据商。

## 预测方法

预测模型是透明启发式评分，不是训练出的机器学习模型。每支球队会从一个中性基础分开始，叠加以下信号：

| 信号 | 作用 |
| --- | --- |
| 历史世界杯四强成绩 | 体现传统强队和淘汰赛经验，并按时间做衰减 |
| World Football Elo | 体现当前国家队公开实力评分 |
| 可选 FIFA 排名/积分 | 作为官方排名补充信号 |
| 当前小组赛场均积分 | 反映本届赛事即时表现 |
| 当前场均净胜球 | 反映攻防差距 |
| 当前场均进球 | 反映进攻表现 |
| 最近三场本届赛事积分 | 反映短期状态 |
| 当前射手贡献 | 根据已完赛进球者给球队轻微加成 |
| 东道主加成 | 2026 年加拿大、墨西哥、美国获得小幅主场加成 |

当前赛事样本会做权重控制。小组赛未踢满前，当前积分、净胜球、进球、状态和射手信号会按已赛场次折扣，避免一场大胜过度主导预测。

## 单场概率如何计算

脚本先计算两队评分差，再通过 logistic 曲线转成胜率。

- 小组赛允许平局，并给接近实力的比赛更高平局概率。
- 淘汰赛不允许平局，因为模拟必须推进晋级球队。

因此下一轮预测会输出队 1 胜、平局、队 2 胜；淘汰赛模拟内部只输出晋级结果。

## 最终排名如何模拟

`predict --target final` 使用蒙特卡洛模拟：

1. 固定已经完赛的小组赛结果。
2. 用单场概率模拟剩余小组赛。
3. 按积分、净胜球、进球数、球队评分给小组排序。
4. 每组前二和成绩最好的 8 个小组第三晋级。
5. 按 2026 赛制解析 32 强淘汰赛占位符。
6. 逐轮模拟 32 强、16 强、8 强、半决赛、三四名决赛和决赛。
7. 统计每队获得冠军、亚军、第三名、第四名的频率。
8. 统计最常见的冠亚季军完整组合。

增加 `--runs` 可以让概率更稳定；设置 `--seed` 可以复现实验结果。

## 输出解读

预测结果应理解为“截至本次抓取数据的方向性概率”。如果数据源尚未记录最新赛果，模型会基于旧状态预测。进行中的赛事建议在回答中说明数据更新时间和来源限制。
